◆ onUnexpectedThrow

template<class Worker >
UnexpectedThrowCallback jaut::BasicLogger< Worker >::Options::onUnexpectedThrow = nullptr

This event occurs when a vital exception was thrown inside a destructor.


This might happen in the log message builder or when the logger gets destructed and need to make a last flush to its sinks.

If no custom handler is used, the program will be terminated as if there was no exception handling. (with additional debug output in debug mode)

If you just want to swallow exceptions, you can supply an empty lambda doing nothing.

◆ logLevel

template<class Worker >
Level jaut::BasicLogger< Worker >::Options::logLevel = JAUT_DEBUGGING_OR(Level::Debug, Level::Info)

The maximum log level, the higher the level the more limited the output is.

For instance, if the log-level is info, all log levels from info downwards will be logged. Or if the log-level is LogLevel::Fatal, this will be the only log that will be sent as it's the highest.

◆ bufferOverflowRetryTimeout

template<class Worker >
int jaut::BasicLogger< Worker >::Options::bufferOverflowRetryTimeout = 0

The amount of milliseconds a log message should be retried to be enqueued when the buffer is full.


This only makes sense when using an async worker or there is an interval specified.

If this is -1, it will block until it could enqueue the message
If this is 0, it will try it once
If this is > 0, it will try it that number of seconds before it gives up

Do note that this might stall the thread forever if it is set to block and any of the flush policies is set unfortunate enough to never flush so be careful with this. (Example: Only mode "Levelled" is enabled and the last message in the buffer happens to not be significant enough of a level to flush the buffer)
