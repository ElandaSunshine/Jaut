Detailed Description

The TabFactory specifies the tab and page behaviour.
This is important to be used as it will determine which id is what page-component and what tabs should look like.

This will also map images to tabs if images are being actively used by this TabFactory object.

If you don't want to use a custom version of this, you can use DefaultTabFactory and pass a lambda with components mapped with ids.

Note that TabFactories are static, you cannot change them afterwards, so you may create your logic inside your TabFactory implementation.

virtual JAUT_NODISCARD Size< int > jaut::MultiPagePanel::AbstractTabFactory::getTabAreaSize ( ) const
pure virtual

Gets the height or width of the tab bar, depending on the TabBarLayout, which determines how big the tab bar should be to contain the tab components.

If this returns 0 or below, it will automatically determine the needed height based on the outermost tab. (position and size of the tab button closest to the ending edge of the tab area)
However, do not that this does not account for whether the tab button is currently visible on screen or not.

If the tab layout is top or bottom, this will determine the height, otherwise the width of the area.

Returns
The size of the tab button area (for both width and height, in case the tab layout changes)

This is where you define the bounds of the tab button.
It will be called whenever a tab is created, or any tab has changed in bounds or was closed.

This is also where you set the layout for the child components of the tab button, but note that you will not always need to change the child components, so make sure to check whether that's really necessary.
